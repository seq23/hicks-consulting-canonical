Quick human instructions

  1. Step 1: Review only the items listed here. If a piece is not shown, it has not passed validation yet.
  2. Step 2: Click Edit in GitHub beside the item you want to change.
  3. Step 3: In the manifest, change the status only:
    ready_for_approval → approved to queue it for release, or published → revoked to pull it down.
  4. Step 4: Commit the GitHub change directly to main.
  5. Step 5: Wait for GitHub Actions to finish. Approved items publish automatically at publishAt. Revoked items disappear on the next build.
  6. Step 6: Refresh this page and the live site to confirm the new status.

Status key

  • ready_for_approval: passed validation and waiting on a human yes
  • approved: approved and waiting for its scheduled publish time
  • published: live on the public site and can be revoked
  • revoked: no longer ships in the public build

Do not change titles, slugs, risk levels, or validation fields here unless you are intentionally editing content ops.
